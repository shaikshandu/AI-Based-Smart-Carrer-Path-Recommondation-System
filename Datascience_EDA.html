Exploratory Data Analysis (EDA) & Visualization

Module 1: Introduction to EDA

  • What is EDA?

    • Exploratory Data Analysis (EDA) is the process of understanding data before building models.

    It helps to: - Detect patterns - Identify anomalies - Understand distributions - Check relationships - Detect missing values

  • Why EDA Matters

    • Most real-world datasets are messy. Without EDA: - Models perform poorly - Insights become misleading - Errors go unnoticed

  • Basic Workflow

    • 1. Load data
    2. Inspect structure
    3. Check missing values
    4. Analyze distributions
    5. Study relationships
    6. Visualize patterns

    Module 2: Descriptive Analysis

  • Central Tendency
    • 
Mean (μ) = (Σ xi) / n
Median = Middle value after sorting
Mode = Most frequent value
  • Spread of Data
    • 
Variance = (Σ (xi - μ)²) / N
Standard Deviation = √Variance
Range = Max - Min
  • Outliers

    • Outliers are extreme values that differ significantly from other observations. 

    
Interquartile Range (IQR):

IQR = Q3 - Q1

Lower Bound = Q1 - 1.5 * IQR
Upper Bound = Q3 + 1.5 * IQR

    Module 3: Data Visualization Fundamentals

  • Why Visualization?

    • Humans understand visuals faster than tables. Visualization transforms numbers into insight.

  • Types of Plots

    • Histogram – Distribution of numerical data
    Bar Chart – Compare categories
    Box Plot – Detect spread and outliers
    Scatter Plot – Relationship between two variables
    Line Chart – Trends over time

  • Distribution Shape

    • Data can be: - Normal - Skewed right - Skewed left - Multimodal 

    
Skewness Formula:

Skewness = (Σ (xi - μ)³ / n) / σ³

    Module 4: Correlation & Relationships

  • Correlation

    • Correlation measures the strength and direction of a relationship between two variables. 

    
Pearson Correlation (r):

r = Σ[(xi - x̄)(yi - ȳ)] 
    --------------------------------
    √(Σ(xi - x̄)² * Σ(yi - ȳ)²)

    r ranges from -1 to +1.

  • Interpretation

    • r = 1 → Perfect positive correlation
    r = -1 → Perfect negative correlation
    r = 0 → No linear relationship

  • Correlation vs Causation

    • Correlation does not imply causation. Two variables moving together does not mean one causes the other.

    Module 5: Feature Engineering & Data Preparation

  • Handling Missing Values

    • Options: - Remove rows - Fill with mean/median - Fill with mode

  • Normalization
    • 
Min-Max Scaling:

X' = (X - Xmin) / (Xmax - Xmin)
  • Standardization
    • 
Z-score Scaling:

Z = (X - μ) / σ
  • Encoding Categorical Data

    • Convert categories into numeric form before model training.

    Final Insight

    EDA is not optional. It is the foundation of trustworthy analysis.

    Good visualization reveals hidden patterns. Poor visualization hides the truth.

    Before building any model, explore the data deeply.

    Strong EDA skills separate beginners from professionals.
