MLOps & Deployment

Module 1: Introduction to MLOps

  • What is MLOps?

    • MLOps (Machine Learning Operations) is the practice of deploying, monitoring, and maintaining machine learning models in production.

    It combines: - Machine Learning - DevOps - Software Engineering

  • Why MLOps is Important

    • Building a model is only 20% of the job. Deploying and maintaining it is the remaining 80%.

    Challenges: - Model versioning - Reproducibility - Monitoring drift - Scaling APIs

    Module 2: Docker Basics

  • What is Docker?

    • Docker is a containerization platform that packages applications with their dependencies.

  • Why Use Docker?

    • Ensures consistency across environments. If it works on your machine, it works everywhere.

  • Dockerfile Example
    • 
FROM python:3.9

WORKDIR /app

COPY requirements.txt .

RUN pip install -r requirements.txt

COPY . .

CMD ["python", "app.py"]
  • Key Commands
    • 
docker build -t model-api .
docker run -p 8000:8000 model-api

    Module 3: Kubernetes Basics

  • What is Kubernetes?

    • Kubernetes orchestrates containers at scale. It manages deployment, scaling, and availability.

  • Core Concepts

    • Pod – Smallest deployable unit
    Deployment – Manages replicas
    Service – Exposes application
    Cluster – Group of nodes

  • Scaling Concept
    • 
Horizontal Scaling:

Increase number of replicas

    Used when traffic increases.

    Module 4: FastAPI for Model Serving

  • Why FastAPI?

    • FastAPI is a high-performance web framework for serving machine learning models as APIs.

  • Simple Model API Example
    • 
from fastapi import FastAPI
import joblib

app = FastAPI()
model = joblib.load("model.pkl")

@app.get("/predict")
def predict(x: float):
    prediction = model.predict([[x]])
    return {"prediction": float(prediction)}
  • API Flow

    • User Request → API Endpoint → Model Prediction → JSON Response

    Module 5: MLflow for Experiment Tracking

  • What is MLflow?

    • MLflow is used to track experiments, log parameters, and manage model versions.

  • Logging Parameters & Metrics
    • 
import mlflow

with mlflow.start_run():
    mlflow.log_param("learning_rate", 0.01)
    mlflow.log_metric("accuracy", 0.92)
  • Why Experiment Tracking Matters

    • Reproducibility ensures: - Same data - Same hyperparameters - Same results

    Module 6: CI/CD for Machine Learning

  • What is CI/CD?

    • Continuous Integration and Continuous Deployment automate testing and deployment pipelines.

  • ML CI/CD Pipeline

    • 1. Code Push
    2. Run Tests
    3. Train Model
    4. Validate Metrics
    5. Deploy Model
    6. Monitor Performance

  • Automation Goal

    • Reduce manual intervention and ensure safe deployments.
